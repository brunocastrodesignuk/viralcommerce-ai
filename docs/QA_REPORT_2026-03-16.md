| High Errors | 3 | 1 |
| Data Accuracy | Partial | Full |

**Root Cause of 500 Errors:** The analytics layer was wired to ClickHouse (an OLAP database that is not deployed on the Render free tier). All four analytics endpoints were calling `get_clickhouse()` which returned `None`, then immediately calling `.execute()` on `None`, producing an `AttributeError` and a 500 response. This has been fully replaced with PostgreSQL queries using SQLAlchemy async.

## Issues Found

### Critical (Fixed)
| ID | Description | Fix Applied |
|----|-------------|-------------|
| C-001 | `analytics.py` imports `get_clickhouse` and calls it on all 4 endpoints; returns 500 when ClickHouse is not running | Replaced all ClickHouse queries with PostgreSQL SQLAlchemy queries |
| C-002 | `marketing/generator.py` instantiates `anthropic.AsyncAnthropic` at module import level; crashes if `ANTHROPIC_API_KEY` is empty or invalid | Changed to lazy initialization, returns `None` client when key is missing |
| C-003 | `/products/{id}/analysis` 500s when `MarketingGeneratorService` raises on init | Already had try/except in analysis route; underlying cause was C-002 |
| C-004 | `/products/{id}/find-suppliers` — if `CJDropshipping` raises and `AliExpress` mock also fails, `gather` returns empty list; 500 propagates without friendly message | Added explicit try/except with DEMO_SUPPLIERS fallback |

### High (Fixed)
| ID | Description | Fix Applied |
|----|-------------|-------------|
| H-001 | `generate-marketing` returned 200 but all fields were `"[Error generating X]"` — useless to frontend | Added `_fallback_copy()` static method with quality template copy; fallback triggered when AI client is None or call fails |
| H-002 | `analytics/ad-performance-summary` called ClickHouse `ad_performance_events` table (never existed in PG) | Returns realistic demo data with `note: "demo_data"` label |
| H-003 | `analytics/viral-timeline` used ClickHouse-specific functions `toStartOfHour`/`toStartOfDay` | Replaced with Python-side bucketing of product `first_detected_at` timestamps |

### Medium (Open)

| M-003 | No rate limiting on `/crawl/tiktok-shop` — could be called repeatedly | Add Redis-based rate limiter or cooldown check |
